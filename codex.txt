Build an MVP job-search automation pipeline in this repository.

Objective:
Create a Google-first pipeline that:
1. ingests job data,
2. stores and organizes it,
3. scores jobs against a candidate profile,
4. generates positioning insights,
5. tailors a master resume,
6. drafts a cover letter,
7. tracks generated artifacts,
8. optionally sends a simple notification when an application packet is ready.

Implementation priorities:
- Favor a working MVP over perfect architecture.
- Keep the code modular, readable, and easy to extend.
- Add clear comments where needed.
- Add a README with setup and usage instructions.
- Add a `.env.example`.
- Do not overbuild.

Technical preferences:
- Primary implementation language: Python.
- Use clean modular structure.
- Prepare integration points for:
  - Google Sheets
  - Google Docs / Drive
  - LLM provider
  - optional Slack/webhook alerts
- For the MVP, where direct Google Docs mutation is too heavy, it is acceptable to first generate structured text outputs and prepare adapters/interfaces for Google integration.
- Use local files / mock adapters where credentials are unavailable, but structure the code so real integrations can be added cleanly.
- Minimize token usage by using rule-based scoring before any model call.

What to build:
A. Repository structure
Create a clear project layout, for example:
- src/
- configs/
- data/
- prompts/
- tests/
- scripts/
- docs/

B. Core modules
Implement modules for:
1. job ingestion
   - accept JSON job input
   - normalize fields
   - compute a stable dedupe key
2. candidate profile loading
   - load a structured candidate profile from YAML or JSON
3. rule-based fit scoring
   - keyword overlap
   - required skill matching
   - title similarity
   - seniority heuristics
   - location / remote checks if present
   - output a normalized score plus reasons
4. shortlist decision
   - configurable threshold
5. insight generation
   - prepare compact structured prompt inputs for an LLM
   - output labels such as:
     - strengths_to_emphasize
     - gaps_to_address
     - summary_angle
     - bullet_focus
6. resume tailoring
   - do not rewrite everything
   - produce constrained edits against a master resume source
   - output:
     - tailored summary
     - bullet edit suggestions
     - keyword insertions
     - optional section ordering suggestions
7. cover letter drafting
   - generate a concise role-specific draft
8. artifact tracking
   - save outputs and metadata locally in a structured way
   - prepare for Google Drive / Sheets syncing later
9. notification adapter
   - simple webhook-based notifier interface

C. Inputs and outputs
Support:
- input job JSON files
- input candidate profile file
- input master resume text/markdown
- output per-job application packet folder containing:
  - normalized_job.json
  - score.json
  - insights.json
  - tailored_resume.md or .txt
  - cover_letter.md or .txt
  - metadata.json

D. Configuration
Add config for:
- shortlist threshold
- scoring weights
- alert toggle
- output directories
- provider placeholders for Google/LLM integration

E. CLI

- run end-to-end on sample data

F. Sample data

Engineering rules:

- Avoid unnecessary dependencies.

- Use dataclasses or pydantic-style models if useful.
- Add basic tests for core scoring logic.
- Make all assumptions visible in code/comments.
